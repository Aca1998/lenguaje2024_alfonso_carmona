Resumen de XML SCHEMA

TEMA 11

XML schema su objetivo es conocer la validacion basada en esquemas XML y saber documentar esquemas

Caracteristicas Principales

  • XML Schema mejora a la validacion DTD.
  • Es un archivo XML.
  • Usa espacios de nombre.
  • Su extension es .XSD

El proposito del esquema XML es definir :

  • Los elementos y atributos que pueden aparecer en un documento
  • El número y orden elementos secundarios
  • Los tipos de datos para elementos y atributos
  • Los valores predeterminados y fijos para elementos y atributos

    • Estructura de un esquema XML (XSD)

  • Definiciones
  • Raiz
  • Elementos
  • Atributos
  • Tipos

    • Tipo de datos para elementos simples

    • String : Secuencia de caracteres.
    • Boolean : Toma los valores true o false.
    • Decimal : Numero decimal con precision arbitraria.
    • Integer : Numero entero positivo o negativo.
    • Date : Fecha en formato "YYYY-MM-DD".
    • Time : Hora en formato "hh:mm:ss".

    Referencias a elementos

  • Declaracion
  • El elemento se define una vez.
  • Se usa el atributo ref.
  • Esto es solo para organizacion del documento.

    • Tipo de datos. Personalizados Simples (limitaciones o restricciones)

    • minInclusive : Minimo valor del numero.
    • maxInclusive : Maximo valor del numero.
    • totalDigits : Numero total de digitos en el numero.
    • length : Nº de caracteres del literal o elementos en lista.
    • pattern : Patron que se tiene que cumplir indicado con expresiones.
